## What I learned today

1. **Multi-agent audits work, but verify before acting.** Agent 5's "auto-
   redirect" P0 was a false alarm — would have wasted 1-2 days hunting a
   non-bug. Direct curl + Playwright check from the orchestrator caught it.

2. **Agents abandon mid-task ~50% of the time on this codebase.** Agent 4
   abandoned the data integrity audit after running schema check + one
   count. Recovery: run the SQL queries myself, write the report inline.
   The `rubli-agent-recovery` skill saved real time.

3. **The "pathetic graphs" complaint was actually about Card chrome,
   not chart components.** Chart inventory: 50% Gold, 46% Decent, 3%
   Pathetic. Time was better spent on Institution/Category Card-grid
   replacement than on more chart polish.

4. **Performance debt surfaces as feature debt.** Categories endpoint
   takes 200-280s under load — pre-existing, not caused by today's
   filter change. But under stress this looks like the filter is broken
   when it's actually the underlying query. Filed as v1.1 candidate.

5. **A loop-killing protocol works when the user enforces it.** Today
   was the first day this session avoided opening new redesigns mid-
   stream. The issue tracker file + "say `Issue #N`" convention means
   we close tickets instead of accumulating them.
