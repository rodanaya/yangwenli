### Critical production bug fixed

The `whileInView` Framer Motion prop was used in 45 chart/story components. This prop relies on IntersectionObserver viewport detection which can fail silently in production when dots are rendered inside SVG elements whose parent container is not yet intersecting. Symptom: dots stay invisible (`opacity: 0`) on first paint even after scrolling into view.

All instances replaced with `animate={{ opacity: 1 }}` (unconditional trigger). Stagger via `transition.delay` preserved. Orphan `viewport={{ once: true }}` props removed.

### Empty dot color standardization

The codebase had three variants of "warm dark" empty dots — `#27272a`, `#2a2420`, `#2a2522` — sometimes within the same page. All dark-context pages (Dashboard, VendorProfile, InstitutionProfile, Administrations, CorruptionClusters, CollusionExplorer, RedThread, YearInReview, ARIA-adjacent) now use the canonical `#2d2926` per ART_DIRECTION §4.

Light-context pages that already used `#f3f1ec` (SpendingCategories light sections, Investigation, DotStrip default, StoryTrenMaya) were left intact. `CorruptionClusters.tsx` had one `backgroundColor: '#f3f1ec'` inside a dark dot strip — left intact since it was an explicit light accent in a small indicator bar.

### Bar chart elimination

Two `<Bar>` elements remained inside `<ComposedChart>` components in `VendorProfile.tsx` (lifecycle chart + rolling stats timeline). Both were bar+line combos showing cumulative value (bars) and risk/win-rate (line). Converted both bars to `<Area>` with `fillOpacity={0.18–0.2}` to preserve the magnitude-by-area signal while matching RUBLI's non-bar aesthetic. Unused `Bar` import removed. Only remaining `<Bar` reference in codebase is a documentation comment in `DotStrip.tsx`.

### OECD reference lines

Both rendered with `stroke="#22d3ee" strokeWidth={1} strokeDasharray="2,2" strokeOpacity={0.55}` and labeled "OCDE 15%" / "OCDE 25%" below the strips. `PAD_B` increased from 18→26 to accommodate the labels.

### Executive Summary — redesign

**Before:** The `/executive` route was a `<Navigate to="/dashboard" replace />`. The page had been removed previously because of "hardcoded stale data + named company risk scores (legal risk)."

**After:** `frontend/src/pages/Executive.tsx` — a new ~460-line editorial 1-pager implementing the ART_DIRECTION §9 spec exactly:

1. **5 numbered key findings** — pulled from `executive.json` `keyFindings5` locale (populated in EN and ES). Each finding has an ordinal (01–05), a hero stat, a label, an editorial headline, and a 2–3 sentence body.
2. **Risk gauge** — custom SVG semicircle arc. Background arc in warm gray; OECD compliance band (2–15%) rendered in cyan at 35% opacity; value arc in red (13.49% exceeds OECD midpoint); animated `pathLength` 0→1 with 1.1s ease; needle + center dot; tick marks at 0/2/15/25; value label in JetBrains Mono 700 28px.
3. **Timeline** — horizontal dot-strip spanning 2002–2025 with 10 landmark corruption cases (IMSS, Estafa Maestra, Oceanografía, Grupo Higa, Odebrecht, Segalmex, COVID, Edenred, Toka, etc). Cases alternate above/below baseline with sector-colored dots; filled = critical, outlined = high. Year ticks at 2002/2006/2012/2018/2024. Motion stagger 50ms.
4. **Single CTA** — amber gold primary button "Open ARIA queue" + secondary outline "Search a vendor" in a tinted card with amber accent border. Uses canonical `#a06820` accent.
5. **Print button** — header action. Inline `@media print` CSS hides sidebar, nav, header, and any `.no-print` elements. Resets background to white, sets `@page { margin: 1.5cm }`, disables shadows.

**Design chrome:**
- Max-width 900px centered
- Playfair Display 800 headline (40–56px responsive): *"1 in 7 pesos of federal spending carries indicators of corruption."*
- Two amber gold gradient dividers between sections
- Credibility strip in footer: `AUC 0.828 · 3.1M contracts · OECD compliant · Open source · RUBLI v0.6.5`
- All numeric values in JetBrains Mono `tabular-nums`
- i18n honored throughout: EN/ES branches for every string

**Data source:** `analysisApi.getFastDashboard()` with safe fallbacks. `risk_distribution` is an array of `{risk_level, count, ...}` so the code reduces it correctly.

**Route wiring:** `App.tsx` — removed `<Navigate to="/dashboard" replace />`, added lazy import of `Executive` and a real Suspense route at `/executive` with `GenericPageSkeleton` fallback.
