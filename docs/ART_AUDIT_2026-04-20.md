#### Patterns found

1. **SVG-level fontFamily inheritance is widespread and correct.** ~13 chart components already set `style={{ fontFamily: FONT_MONO }}` on the outer `<svg>` element. This causes children `<text>` elements to inherit mono correctly without per-element `fontFamily=` props. 2. **Section badge drift.** Two different badge patterns coexisted: the canonical `text-[10px] font-semibold uppercase tracking-widest` (dominant — 30+ files) and a non-canonical `text-xs uppercase tracking-[0.2em] text-text-muted font-semibold` (10 occurrences in 2 files). 3. **Font-serif on numeric values — already eliminated.** The prior TIPO pass fixed `HallazgoStat.tsx`. A fresh grep confirmed zero remaining cases where Playfair Display wraps a numeric JSX expression.
4. **Zero body-prose font-mono violations.** Grep for `text-(base|lg) ... font-mono ... leading-relaxed` returned no matches, confirming Cardinal Rule #3 holds.
5. **Zero hero-size number violations.** Grep for `text-(4xl|5xl|6xl)` wrapping numeric expressions without `font-mono`/`tabular-nums` returned no matches at the page level. The one `text-3xl` violation in `Settings.tsx` was fixed.

#### Remaining manual review needed

- **~370 `rounded-xl` / `rounded-2xl` / `rounded-lg` occurrences across ~147 files.** Prior TIPO pass note claimed zero remain, but a fresh grep counts 370+. These are mostly on chart tooltips, pill/badge containers, dialogs, modals, and cinematic story charts. Cardinal Rule #6 specifies `rounded-sm` for cards but allows non-sm on buttons/badges/pills. A blanket mass-replace would regress UX on dialogs/tooltips/inputs where `rounded-lg` is the Shadcn default. Recommend a per-component judgment pass — approximately 80–120 of these are truly "card container" violations. Low-risk pattern: audit `card-elevated`, `fern-card`, `surface-card` references first (these are the semantic "cards" by name).
- **`tracking-[0.2em]` vs `tracking-widest` (0.1em) drift.** 30+ files use `tracking-[0.2em]` on section badges. ### Critical production bug fixed

The `whileInView` Framer Motion prop was used in 45 chart/story components. This prop relies on IntersectionObserver viewport detection which can fail silently in production when dots are rendered inside SVG elements whose parent container is not yet intersecting. Symptom: dots stay invisible (`opacity: 0`) on first paint even after scrolling into view.

All instances replaced with `animate={{ opacity: 1 }}` (unconditional trigger). Stagger via `transition.delay` preserved. Orphan `viewport={{ once: true }}` props removed.

### Empty dot color standardization

The codebase had three variants of "warm dark" empty dots — `#27272a`, `#2a2420`, `#2a2522` — sometimes within the same page. All dark-context pages (Dashboard, VendorProfile, InstitutionProfile, Administrations, CorruptionClusters, CollusionExplorer, RedThread, YearInReview, ARIA-adjacent) now use the canonical `#2d2926` per ART_DIRECTION §4.

Light-context pages that already used `#f3f1ec` (SpendingCategories light sections, Investigation, DotStrip default, StoryTrenMaya) were left intact. `CorruptionClusters.tsx` had one `backgroundColor: '#f3f1ec'` inside a dark dot strip — left intact since it was an explicit light accent in a small indicator bar.

### Bar chart elimination

Two `<Bar>` elements remained inside `<ComposedChart>` components in `VendorProfile.tsx` (lifecycle chart + rolling stats timeline). Both were bar+line combos showing cumulative value (bars) and risk/win-rate (line). Converted both bars to `<Area>` with `fillOpacity={0.18–0.2}` to preserve the magnitude-by-area signal while matching RUBLI's non-bar aesthetic. Unused `Bar` import removed. Only remaining `<Bar` reference in codebase is a documentation comment in `DotStrip.tsx`.

### OECD reference lines
